网络编程学习笔记

一、网络编程概述

  • 网络编程：在网络通信协议下，不同计算机上运行的程序，进行的数据传输。

  • 不管是什么场景，都是计算机跟计算机之间通过网络进行数据传输。

  • Java 中可以使用 java.net 包下的技术轻松开发出常见的网络应用程序。

  • 常见的软件架构

    • C/S：Client/Server 客户端/服务器

      在用户本地需要下载并安装客户端程序，在远程有一个服务器端程序。

    • B/S：Browser/Server 浏览器/服务器

      只需要一个浏览器，用户通过不同的网址访问不同的服务器。

  • 两种软件架构的区别：

    • CS：适合定制专业化的办公类软件，如：IDEA、网游；
    • BS：适合移动互联网应用，可以在任何地方随时访问的系统。

二、网络编程三要素

1. IP

  • 全称：Internet Protocol，是互联网协议地址，也称 IP 地址。是分配给上网设备的数字标签，是唯一的标识。

  • IPv4：目前的主流方案，最多只有 2322^{32}232 个 IP，目前已经用完了。

  • IPv6：为了解决 IPv4 不够用而出现的，最多有 21282^{128}2128 个 IP。

  • 获取 IP 对象

2. 端口号

  • 应用程序在设备中唯一的标识。

  • 由两个字节表示的整数，取值范围：0~65535。

  • 其中 0~1023 之间的端口号用于一些知名的网络服务或应用。

  • 一个端口号只能被一个应用程序使用。

3.协议

  • 数据在网络中传输的规则，常见的协议有 UDP, TCP, http, ftp。
  • UDP 协议
    • 用户数据报协议（User Datagram Protocol）
    • UDP 是面向无连接通信协议。
    • 速度快，有大小限制，一次最多发送 64K，数据不安全，易丢失数据。
  • TCP 协议
    • 传输控制协议（Transmission Control Protocol）
    • TCP 协议是面向连接的通信协议。
    • 速度慢，没有大小限制，数据安全。

1. 发送数据

  1. 创建发送端的 DatagramSocket 对象
  2. 数据打包 DatagramPacket
  3. 发送数据
  4. 释放资源

2. 接收数据

  1. 创建接收端的 DatagramSocket 对象
  2. 接收打包好的数据
  3. 解析数据包
  4. 释放资源

3. UDP 的三种通信方式

  1. 单播

    • 之前的代码都是单播

  2. 组播

    • 组播地址：240.0.0.0~239.255.255.255

    • 其中 224.0.0.0~224.0.0.255 为预留的组播地址

  3. 广播

    • 广播地址：255.255.255.255

四、TCP 通信程序

  • TCP 通信协议是一种可靠的网络协议，它在通信的两端各建立一个 Socket 对象
  • 通信之前要保证连接已经建立
  • 通过 Socket 产生 IO 流来进行网络通信

1. 发送数据

  1. 创建客户端的 Socket 对象（Socket）与指定服务器连接

  2. 获取输出流，写数据

  3. 释放资源

2. 接收数据

  1. 创建服务器端的 Socket 对象（ServerSocket）

  2. 监听客户端连接，返回一个 Socket 对象

  3. 获取输入流，读数据，并把数据显示在控制台

  4. 释放资源
