Java IO流学习笔记

发表于2024-07-28|更新于2024-07-30|java 学习笔记
|字数总计:3.9k|阅读时长:15分钟|阅读量:

一、IO 流概述

  1. 什么是 IO 流？

    存储和读取数据的解决方案

    I：Input O：Output

    流：像水流一样传输数据

  2. IO 流的作用

    用于读写数据（本地文件，网络）

  3. IO 流按照流向可以分为哪两种流？

    输出流：程序 -> 文件

    输入流：文件 -> 程序

  4. IO 流按照操作文件的类型可以分类成哪两种流？

    字节流：可以操作所有类型的文件

    字符流：只能操作纯文本文件

  5. 什么是纯文本文件

    用 Windows 系统自带的记事本打开并且能读懂的文件

    txt 文件、md 文件、xml 文件、lrc 文件等。

二、IO 流体系

  • 字节流
    • java.io.InputStream 字节输入流
    • java.io.OutputStream 字节输出流
  • 字符流
    • java.io.Reader 字符输入流
    • java.io.Writer 字符输出流
  • 注：
    • 四个类都是抽象类；
    • 所有的流都实现了 java.io.Closeable 接口，都是可关闭的，都有 close() 方法。流是一个管道，这个是内存和硬盘之间的通道，用完后一定要关闭，不然会耗费很多资源。
    • 所有的输出流都实现了 java.io.Flushable 接口，都是可刷新的，都有 flush() 方法。输出流在最终输出后，最好用 flush() 刷新一下。这个刷新表示将通道/管道当中剩余未输出的数据强行输出完，刷新的作用就是清空管道。（如果没有 flush() 可能会导致丢失数据）
    • 在 Java 中只要类名以 Stream 结尾的都是字节流。以 Reader/Writer 结尾的都是字符流。

三、基本流

1. 字节输入流 FileInputStream

任何类型的文件都可以采用这个流来读。

  • 构造方法

    构造方法名 备注
    FileInputStream(String name) name 为文件路径
    FileInputStream(File file)

  • 方法

    方法名 作用
    int read() 读取一个字节，返回值为该字节 ASCII 码；读到文件末尾返回 -1
    int read(byte[] b) 读 b 数组长度的字节到 b 数组中，返回值为读到的字节个数；读到文件末尾返回 -1
    int read(byte[] b, int off, int len) 从 b 数组 off 位置读 len 长度的字节到 b 数组中，返回值为读到的字节个数；读到文件末尾返回 -1
    int available() 返回文件有效的字节数
    long skip() 跳过 n 个字节
    void close() 关闭文件输入流

2. 字节输出流 FileOutputStream

  • 构造方法

    构造方法名 备注
    FileOutputStream(String name) name 为文件路径
    FileOutputStream(String name, boolean append) name 为文件路径，append 为 true 表示在文件末尾追加；为 false 表示清空文件内容，重新写入
    FileOutputStream(File file)
    FileOutputStream(File file, boolean append) append 为 true 表示在文件末尾追加；为 false 表示清空文件内容，重新写入

  • 方法

    方法名 作用
    void write(int b) 将指定字节写入文件中
    void write(byte[] b) 将 b.length 个字节写入文件中
    void write(byte[] b, int off, int len) 将 b 数组 off 位置开始，len 长度的字节写入文件中
    void flush() 刷新此输出流并强制写出所有缓冲的输出字节
    void close() 关闭文件输出流

  • 与字节输入流结合实现拷贝文件操作

3. 字符集

  1. 在计算机中，任意数据都是以二进制的形式来存储的。计算机中最小的存储单元是一个字节。

  2. ASCII 字符集中，一个英文占一个字节。

  3. GBK 字符集完全兼容 ASCII 字符集：

    1. 一个英文占一个字节，二进制第一位是 0；
    2. 一个中文占两个字节，二进制高位字节的第一位是 1.

  4. Unicode 字符集，也称万国码，其中 UTF-8 编码规则为：

    二进制形式 文字类型
    0xxxxxxx ASCII 码
    110xxxxx 10xxxxxx 叙利亚文
    1110xxxx 10xxxxxx 10xxxxxx 中东文字（包括中国）
    11110xxx 10xxxxxx 10xxxxxx 10xxxxxx 其他

4. 字符输入流 FileReader

文件字符输入流，只能读取纯文本。读取纯文本内容时，比较方便、快捷。

  • 构造方法

    构造方法名 备注
    FileReader(String Name) Name 为文件路径
    FileReader(File file)

  • 方法

    方法名 作用
    int read() 读取一个字符，返回值为该字符编码；读到文件末尾返回 -1
    int read(char[] c) 读 c 数组长度的字符到 c 数组中，返回值为读到的字符个数；读到文件末尾返回 -1
    int read(char[] c, int off, int len) 从 c 数组 off 位置读 len 长度的字符到 c 数组中，返回值为读到的字符个数；读到文件末尾返回 -1
    long skip(long n) 跳过 n 个字符
    void close() 关闭文件输入流

5. 字符输出流 FileWriter

文件字符输出流，只能输出普通文本。

  • 构造方法

    构造方法名 备注
    FileWriter(String name) name 为文件路径
    FileWriter(String name, boolean append) name 为文件路径，append 为 true 表示在文件末尾追加；为 false 表示清空文件内容，重新写入
    FileWriter(File file)
    FileWriter(File file, boolean append) append 为 true 表示在文件末尾追加；为 false 表示清空文件内容，重新写入

  • 方法

    方法名 作用
    void write(int c) 将指定字符写入文件中
    void write(char[] c) 将 c.length 个字符写入文件中
    void write(char[] c, int off, int len) 将 c 数组 off 位置开始，len 长度的字符写入文件中
    void write(String str) 将字符串写入文件中
    void write(String str, int off, int len) 从字符串 off 位置开始截取 len 长度的字符串写入文件
    void flush() 刷新此输出流并强制写出所有缓冲的输出字符
    void close() 关闭文件输出流

四、缓冲流

​ 高级流，创建时需要传入特定的基本流，因为底层是带有缓冲区的基本流，能大幅提高读写效率，包括字节缓冲输入流 BufferedInputStream、字节缓冲输出流 BufferedOutputStream、字符缓冲输入流 BufferedReader、字符缓冲输出流 BufferedWriter。其中方法大多与基本流中的方法相同，但是 BufferedReader 和 BufferedWriter 中各有 readLine() 和 newLine() 方法是特有的，并且十分常用。

1. 字符缓冲输入流 BufferedReader

BufferedReader 带有缓冲区的字符输入流，包装了基本流 Reader。

  • 构造方法

    构造方法名 备注
    BufferedReader(Reader in) in 为 Reader 对象（Reader 的实现类）

  • 方法

    方法名 作用
    int read() 读取一个字符，返回值为该字符编码；读到文件末尾返回 -1
    int read(char[] c) 读 c 数组长度的字符到 c 数组中，返回值为读到的字符个数；读到文件末尾返回 -1
    String readLine() 读取文件一行（不包括换行符）；若读不到数据返回 null
    long skip(long n) 跳过 n 个字符
    void close() 关闭文件输入流

2. 字符缓冲输出流 BufferedWriter

BufferedWriter 带有缓冲的字符输出流。

  • 构造方法

    构造方法名 备注
    BufferedWriter(Writer out) out 为 Writer 对象（Writer 的实现类）

  • 方法

    方法名 作用
    void write(int c) 将指定字符写入文件中
    void write(char[] c, int off, int len) 将 c 数组 off 位置开始，len 长度的字符写入文件中
    void write(String str, int off, int len) 从字符串 off 位置开始截取 len 长度的字符串写入文件
    void newLine() 写入一个行分隔符。行分隔符字符串由系统属性 line.separator 定义，并且不一定是单个新行 (’\n’) 符。
    void flush() 刷新次输出流并强制写出所有缓冲的输出字符
    void close() 关闭文件输出流

五、转换流

字节流和字符流之间的桥梁。

  • 作用：
    1. 指定字符集读写。（jdk11 后被淘汰了）
    2. 字节流要使用字符流中的方法。

1. 字节输入流转字符输入流 InputStreamReader

将字节输入流转换为字符输入流。

  • 构造方法

    构造方法名 备注
    InputStreamReader(InputStream in) 创建一个使用默认字符集的 InputStreamReader
    InputStreamReader(InputStream in, Charset cs) 创建使用给定字符集的 InputStreamReader
    InputStreamReader(InputStream in, String charsetName) 创建使用指定字符集的 InputStreamReader

2. 字符输出流转字节输出流 OutputStreamWriter

将字符输出流转换为字节输出流。

  • 构造方法

    构造方法名 备注
    OutputStreamWriter(OutputStream out) 创建使用默认字符编码的 OutputStreamWriter
    OutputStreamWriter(OutputStream out, Charset cs) 创建使用给定字符集的 OutputStreamWriter
    OutputStreamWriter(OutputStream out, String charsetName) 创建使用给定字符集的 OutputStreamWriter

六、序列化流

  • 前言：

    • 参与序列化和反序列化的对象必须实现 Serializable 接口。

    • Serializable 接口是一个标识接口，其中没有任何代码，作用：

      • 起到标识作用，Java 虚拟机看到这个类实现了这个接口，可能会对这个类进行特殊待遇；
      • Serializable 这个标识接口是给 Java 虚拟机参考的，Java 虚拟机看到这个接口后，会为该类自动生成一个序列化版本号。

    • 序列化版本号

      • 写法：

      • 作用：区分两个类是否相同。在 Java 中，首先对类名进行比对，如果不同，肯定不是一个类；如果相同，通过序列化版本号进行区分。凡是提供了固定不变的序列化版本号的类，即使类中代码发生了改变，Java 虚拟机依然认为是同一个类。

      • 怎样使某个属性不序列化？

        使用 transient 关键字，表示游离的，不参与序列化。

1. 序列化流 ObjectOutputStream

可以将 Java 中的对象写到本地中。

  • 构造方法

    构造方法名 备注
    ObjectOutputStream(OutputStream out) out 为 OutputStream 对象

  • 常用方法

    方法名 作用
    void writeObject(Object obj) 将指定的对象写入 ObjectOutputStream

  • 由于反序列化流读取数据至文件结尾时会直接抛出异常，因此除非在将来读取时明确知道对象的数量，否则一般推荐将对象放到集合中再序列化。

2. 反序列化流 ObjectInputStream

可以把序列化到本地文件中的对象，读取到程序中来。

  • 构造方法

    构造方法名 备注
    ObjectInputStream(InputStream in) in 为 InputStream 对象

  • 常用方法

    方法名 作用
    Object readObject() 从 ObjectInputStream 读取对象

七、打印流

PrintStream，标准的字节输出流。默认输出到控制台。

  • 构造方法

    构造方法名 备注
    PrintStream(File file)
    PrintStream(OutputStream out)
    PrintStream(String fileName) fileName 文件地址

  • 方法

    方法名 作用
    println(参数类型不定 x) 输出 x 并换行
    print(参数类型不定 x) 输出 x
    void flush() 刷新此输出流并强制写出所有缓冲的输出字符
    void close() 关闭流

八、文件

  • File 类和 IO 流没有关系，所以 File 类不能完成文件的读和写；

  • File 对象是文件和目录路径名的抽象表示形式。

  • 构造方法

    构造方法名 备注
    File(String pathName) pathName 文件/文件夹路径

  • 方法

    方法名 作用
    boolean delete() 删除文件/文件夹
    boolean exists() 判断文件/文件夹是否存在
    File getAbsoluteFile() 获取文件/文件夹的绝对路径
    String getName() 获取文件/文件夹名字
    File getParent() 获取文件/文件夹的父文件/文件夹
    String getPath() 获取文件/文件夹的路径
    boolean isDirectory() 判断该文件/文件夹是不是文件夹
    boolean isFile() 判断该文件/文件夹是不是文件
    isHidden() 判断该文件/文件夹是否被隐藏
    long lastModified() 获取文件/文件夹最后一次修改时间
    long length() 获取文件大小；获取文件夹里面的文件个数
    String[] list() 获取文件夹里的文件名字并以 String[] 返回
    File[] listFiles() 获取文件夹里的文件名字并以 File[] 返回
    boolean mkdir() 创建文件夹（父级路径必须存在）
    boolean mkdirs() 创建多重文件夹
    boolean createNewFile() 当且仅当不存在具有此抽象路径名指定的文件时，不可分地创建一个新的文件
