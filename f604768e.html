Tools原理和注意事项

  • 若响应成功，大模型会在响应体中返回需要使用的方法名称和入参：

二、Tools 注意事项

1. 参数或返回值不支持

  • 不支持的参数或返回类型：
  • Optional
  • 异步类型（CompletableFuture 例如：Future）
  • 反应类型（例如：Flow、Mono、Flux）
  • 功能类型（例如：Function、Supplier、Consumer）
  • 推荐：pojo、record、java基础类型、list、map

2. 温度过低导致参数无法自动推算

  • 温度（模型随机性）太低，AI 可能缺失自由度变得比较拘谨
  • 可以通过描述使其更明确

3. 大模型“强行适配”Tool 参数导致幻觉

  • 加严参数描述与校验
  • 后端代码加强校验和兜底保护
  • 系统 Prompt 设定限制
  • 高风险接口（资金、风控等）tools 方法加强人工确认，多走一步校验

4. 工具暴露的接口名、方法名、参数名要可读、业务化

  • AI 是“看”你的签名和注释来决定用不用工具的
  • 尽量避免乱码、缩写等

5. 方法参数数量不宜过多

  • 建议每个工具方法尽量少于 5 个参数，否则 AI 提示会变复杂，出错率高

6. 工具方法性能要高

  • 工具方法不适合做超耗时操作，更长的耗时意味着用户延迟响应时间变长

7. 关于 Tools 的权限控制

1）利用 SpringSecurity 限制

2）将 tools 和权限资源一起存储，然后动态设置 tools

8. tools 过多导致 AI 出现选择困难症

  • tools 过多会导致两个问题：
  • 触发 token 上线
  • 选择困难症
  • 解决办法：使用向量数据库 RAG，做相似性检索：
  1. 把所有的 tools 描述信息存入到向量数据库
  2. 每次对话的时候根据当前对话信息检索到相似的 tools
  3. 动态设置 tools
